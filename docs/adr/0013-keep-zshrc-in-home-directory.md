# ADR-0013: .zshrc をホームディレクトリに維持する

## ステータス
採用

## コンテキスト
XDG Base Directory Specification に従い、`.zshrc` を `~/.config/zsh/.zshrc` に移動することを検討した。

zsh は `ZDOTDIR` 環境変数を使って設定ファイルの場所を変更でき、`~/.config/zsh/` 以下にまとめることが可能。ただし zsh は公式には XDG に対応しておらず（2013年から議論されているが未対応）、ブートストラップ用の `~/.zshenv` だけはホームディレクトリに残す必要がある。

## 決定
`.zshrc` は従来通り `~/.zshrc`（`config/.zshrc`）に置く。XDG 準拠への移行は行わない。

## 理由
- zsh 自体が公式に XDG をサポートしていない
- `~/.zshenv` をホームに残す必要があり、完全にはクリーンにならない
- 多くのツールやドキュメントが `~/.zshrc` を前提としている
- 移行に伴うリスク（パスの書き換え漏れ等）に対してメリットが小さい

## 結果
- `config/.zshrc` を home-manager で `~/.zshrc` にリンクする現行方式を維持する
- 将来 zsh が公式に XDG 対応した場合は再検討する
