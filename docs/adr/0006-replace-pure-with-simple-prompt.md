# ADR-0006: Pure プロンプトをシンプルなカスタムプロンプトに置き換え

## ステータス
採用

## コンテキスト
Pure プロンプトを使用していたが、シェルの起動が遅くなる問題があった。
特に大きな git リポジトリや初回起動時に数秒かかることがあった。

Pure は非同期で git の状態を取得する機能があるが、それでも起動時間に影響していた。

## 決定
Pure プロンプトを削除し、シンプルなカスタムプロンプトに置き換える。

## 理由
- **起動速度**: Pure の git fetch やその他の機能が起動を遅くしていた
- **シンプルさ**: git ブランチ名の表示だけで十分
- **依存の削減**: homebrew の pure パッケージが不要になる

## 結果

### 削除
- `darwin/homebrew.nix` から `pure` を削除

### 変更
- `config/zsh/init/cmd/pure.sh` → `config/zsh/init/cmd/prompt.sh` にリネーム
- プロンプトをシンプルな実装に変更

### 新しいプロンプト
```
~/dotfiles (main) %
```

- カレントディレクトリ（青）
- git ブランチ名（黄色、git リポジトリ内のみ）
- プロンプト記号
