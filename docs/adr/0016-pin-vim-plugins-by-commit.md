# ADR-0016: vim プラグインを commit hash で固定する

## ステータス

採用

## コンテキスト

`.vimrc` では vim-plug で 31 個のプラグインを管理しているが、すべてブランチ追従
(デフォルトで master/main の HEAD)になっており、`:PlugUpdate` を実行すると
upstream の任意の最新コミットがそのまま取り込まれる状態だった。

近年、エディタプラグインやパッケージレジストリを標的とした supply chain 攻撃
(メンテナアカウントの乗っ取り、リポジトリの譲渡先による悪意ある変更など)が
増えており、特に長期間更新されていないプラグインはリスクが高い。

## 決定

vim-plug の `commit` オプションを使い、全プラグインを commit hash で固定する。

```vim
Plug 'rhysd/clever-f.vim', { 'commit': '6a3ac5e3688598af9411ab741737f98c47370c22' }
```

固定するハッシュは、決定時点でローカルにインストール済みだった
(= 動作確認済みの)各プラグインの HEAD を採用した。

## 理由

- `:PlugUpdate` を実行しても未検証のコードが入ってこなくなり、
  upstream が侵害されても影響を受けない
- どのバージョンを使っているかが `.vimrc` に明示され、git で追跡できる
- プラグインの更新は「ハッシュを書き換えてコミットする」という
  明示的なレビュー可能な操作になる

## 結果

- プラグインを更新する場合は、upstream の変更内容を確認した上で
  `.vimrc` のハッシュを書き換えてから `:PlugUpdate` を実行する
- 自動的に最新版へ追従しなくなるため、セキュリティ修正も自動では入らない。
  必要に応じて手動で更新する
- 今後の課題: vim-plug 本体 (`~/.vim/autoload/plug.vim`) は curl で
  master から取得する手順のままなので、リポジトリへの vendoring を検討する。
  また、使っていないプラグインの削減も別途検討する
