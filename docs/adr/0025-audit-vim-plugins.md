# ADR-0025: vim プラグインのソースコードを監査する

## ステータス

採用

## コンテキスト

ADR-0016 で全 vim プラグインを commit hash で固定し、ADR-0024 で 10 個まで
削減した。commit 固定は「pin した時点の内容が以後勝手に変わらない」ことを
保証するが、「pin した内容そのものが安全か」は別問題である。

供給元 (GitHub リポジトリ) が pin 時点で既に侵害されていた場合、commit を
固定しても悪意あるコードを取り込んでしまう。そこで、固定している 10 個の
プラグインのソースコードを実際に監査し、安全性を確認することにした。

## 決定

全 10 プラグインを supply chain security の観点で監査し、結果を
`config/.vim/AUDIT.md` に記録する。

監査観点: ネットワーク通信 / 任意コード実行 / modeline・autocmd 経由の自動実行 /
難読化 / 情報送信 / バイナリの有無 / 既知の危険パターン (コマンドインジェクション等)。

各プラグインを個別に精査し、事前に実体の HEAD が `.vimrc` の pin と一致する
ことも確認した。

## 結果

**全 10 プラグインが CLEAN。** バックドア・悪意あるコード・難読化・意図しない
外部通信・情報窃取の痕跡はいずれにも認められなかった。詳細な所見は
`config/.vim/AUDIT.md` を参照。

- 外部コマンドを呼ぶ箇所 (vim-go の go/gopls 等、ctrlp の find/ag/ctags、
  vim-perl・perl-local-lib-path の perl 実行) はいずれも正当な機能であり、
  ユーザーの明示的操作またはローカル環境情報の取得に限定され、引数も
  shellescape 済みであることを確認した
- editorconfig-vim については、悪意ある `.editorconfig` から vim へコードを
  注入できないか (信頼境界) を重点的に確認し、値が `execute`/`eval`/`system` に
  渡る経路がないことを確認した
- 軽微な堅牢性バグ (perl-local-lib-path の `path` 未エスケープ、quickfixstatus の
  `\n` substitute) は見つかったが、いずれもセキュリティ上の問題ではない

## 今後

- プラグインを更新 (ハッシュ書き換え) する際は、upstream の差分をレビューした
  上で、必要に応じて該当プラグインを再監査し AUDIT.md を更新する
- 新しいプラグインを追加する際も、commit 固定 (ADR-0016) と合わせて監査する
